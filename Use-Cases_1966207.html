Travel Booking System : Use Cases

Created by Jwalit Chiragbhai Shah, last modified by Anushka Shetty on Jun 09, 2023

Main use cases with the basic flow and alternative flows

Sprint 1:

Use Case 1: Search Packages 

Actor: Customer

Basic Flow:

  1. The customer enters their desired search criteria, such as destination.

  2. The system retrieves and displays a list of packages that match the search criteria.

Alternative Flow:

      A1. There are no packages available that match the search criteria

Use Case 2: Create Package

Actor: Agent

Basic Flow:

  1. The agent selects the option to create a new package.

  2. The system presents a form for the agent to enter package details, such as destination, duration, and price.

  3. The agent fills in the required information and submits the form.

  4. The system saves the package details.

Alternative Flow:

      A1. Required fields are not filled

      A2. The same package already exists

Use Case 3: View Available Packages

Actor: Customer

Basic Flow:

  1. The customer selects the option to view available packages.

  2. The system retrieves and displays a list of all available packages.

Alternative Flow:

      A1. There are no travel packages created yet

Use Case 4: Select Flights

Actor: Customer

Basic Flow:

  1. The customer is creating a custom package.

  2. The customer selects the flight option.

  3. The system presents a list of available flights.

  4. The customer selects the desired flights and adds them to their package.

Alternative Flow:

      A1. The list of available flights is not shown

      A2. Field validation is not satisfied while custom package creation

      A3. Flight rate is not pre-filled after flight selection

Use Case 5: Select Hotels

Actor: Customer

Basic Flow:

  1. The customer is creating a custom package.

  2. The customer selects the hotel's option.

  3. The system presents a list of available hotels.

  4. The customer selects the desired hotels and adds them to their package.

Alternative Flow:

      A1. The list of available hotels is not shown

      A2. Field validation is not satisfied while custom package creation

      A3. The hotel rate is not pre-filled after the hotel selection

Use Case 6: Select Activities

Actor: Customer

Basic Flow:

  1. The customer is creating a custom package.

  2. The customer selects the activities option.

  3. The system presents a list of available activities.

  4. The customer selects the desired activities and adds them to their package.

Alternative Flow:

      A1. The list of available activities is not shown

Use Case 7: View Total Cost

Actor: Customer

Basic Flow:

  1. The customer is creating a custom package.

  2. The system dynamically calculates and displays the total cost of the package based on the selected flights, hotels, and activities.

Alternative Flow:

      A1. The wrong cost is calculated

Use Case 8: Store Custom Packages

Actor: Agent

Basic Flow:

  1. The agent is creating a custom package on behalf of a customer.

  2. The agent fills in the required details for the custom package, such as flights, hotels, and activities.

  3. The agent saves the custom package in the system.

Alternative Flow:

      A1. Field validation fails 

      A2. The same custom package already exists

      A3. Server is crashed

Use Case 9: Create a User Account

Actor: Customer and Agent (User)

Basic Flow:

  1. The user selects the option to create a new account.

  2. The system presents a registration form.

  3. The user fills in the required information and submits the form.

  4. The system creates a new user account.

Alternative Flow:

      A1. User already exists

      A2. Input field validation fails

      A3. Server is crashed

Use Case 10: User Login

Actor: Customer and Agent (User)

Basic Flow:

  1. The user enters their login credentials.

  2. The system verifies the credentials.

  3. The system logs in to the user and displays the user's dashboard.

Alternative Flow:

      A1. Incorrect username or password

      A2. Empty fields

      A3. Server not responding

      A4. The local storage cache is not functional due to browser issues

Sprint 2:

User case 1: Create a Booking

Slice 1: Show all the bookings of the logged user 

Basic flow:

  1. The system filters the bookings made by the logged customer

  2. The system fetches all bookings in case of agents

  3. The system presents the already created bookings to the user with all data

Slice 2: Select the available travel package from the available travel package dropdown

Basic flow:

  1. The system filters the bookings made by logged customers and all bookings in the case of agents

  2. The system presents the already created bookings to the user with all data

Slice 3: Validate the changes for the input fields

Basic flow:

  1. The system validates the input fields with their validation rules

  2. The system shows an error message on the fields where the incorrect input is given

  3. The system does not execute further action until valid input is provided

Use case 2: Modify Booking

Slice 1: Select booking from the already made booking dropdown

Basic flow:

  1. The system filters the bookings made by logged customers and all bookings in the case of agents

  2. The system shows a list of bookings in the dropdown menu

  3. The user selects the booking id

Slice 2: Show all the bookings details to the customer

Basic flow:

  1. The user selects the booking id

  2. The system fetches the booking data from the database

  3. The system shows the booking information prefilled in the input fields

Slice 3: Validate the changes for the input fields

Basic flow:

  1. The system validates the input fields with their validation rules

  2. The system shows an error message on the fields where the incorrect input is given

  3. The system does not execute further action until valid input is provided

Slice 4: Update the existing booking with new data

Basic flow:

  1. The system updates the existing booking

  2. The system shows the message came from the server

Use case 3: Send Notifications

Slice 1: Send notifications for the desired user actions

Basic flow:

  1. The user performs an action that notifies the user of the successful completion

  2. The system sends an email to the user with their provided email id while signup

  3. The user receives the email notification

Use case 4: View reports

Slice 1: Agent Selects the option to view reports

Basic flow:

  1. The agent selects a button to view the reports

  2. The system will check the user type

Slice 2: System generates and displays reports on customer bookings and revenue

Basic flow:

  1. The system fetches all the revenue generated from the available travel packages through the bookings

  2. The system displays the report to the user

Use case 5: Manage User Accounts

Slice 1: User selects the update profile button

Basic flow:

  1. The user selects the update profile button

  2. The system fetches the user data from the local storage

  3. The system shows the profile information prefilled in the input fields

Slice 2: Update existing information with proper input validations

Basic flow:

  1. The system validates the input fields with their validation rules

  2. The system shows an error message on the fields where the incorrect input is given

  3. The system does not execute further action until valid input is provided

Slice 3: Update user profile

Basic flow:

  1. The system updates the logged user information

  2. The system shows the message came from the server
