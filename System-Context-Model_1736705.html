Customer

The customer role in the travel booking system refers to individuals who are looking to book touristic packages for their travels. They access the system either through a browser from any location.

Customers can access the system to add, view and select touristic packages. They can search for available packages, view details such as destination, duration, hotel information, activities included, and prices. Customers can also view their bookings, modify or cancel them as necessary.

Customers can interact with the system as per their needs. They can browse and search for packages at their preferred frequency. The volume of data exchanged would depend on the specific actions taken by the customer, such as searching for packages, viewing package details, or managing bookings.

Agent

The agent role in the travel booking system refers to travel agents who are responsible for managing packages, bookings, customer profiles, and generating reports. They utilize the system to create and customize touristic packages, handle customer bookings, maintain customer profiles, and perform reporting tasks. Agents can access the system through a browser.

Agents require access to functionalities and information such as creating and managing touristic packages, managing customer profiles, making bookings for customers, modifying or canceling bookings, and generating reports on bookings and revenue.

Agents can interact with the system as needed. The frequency and volume of data exchanged would depend on the specific actions taken by agents, such as creating or modifying packages, managing bookings and customer profiles, and generating reports.

Payment Gateway

The Payment Gateway is an external service integrated into the travel booking system that handles the processing and authorization of online payments. It facilitates secure and seamless transactions between customers and the system, ensuring the confidentiality and integrity of payment information.

Data Format: The Payment Gateway typically utilizes structured data formats for transmitting payment-related information.

Access Protocol:

  • HTTPS: The system interacts with the Payment Gateway using HTTPS protocols. It sends payment requests and receives responses over secure connections.

  • API : The Payment Gateway exposes a set of well-defined APIs that the travel booking system can use to initiate payment transactions.

Compatibility: The Payment Gateway supports a wide range of payment methods, including credit cards, debit cards, digital wallets.

Performance: The system is optimized for fast and efficient payment processing. It minimizes transaction latency, handles high volumes of concurrent transactions.

Integration: The Payment Gateway offers seamless integration capabilities with the travel booking system. It provides well-documented APIs (Application Programming Interfaces) or SDKs (Software Development Kits) that allow the system to securely communicate with the gateway, exchange payment-related data, and receive transaction status updates.

Authentication

The Authentication System is responsible for verifying the identity of users and granting access to the travel booking system based on their credentials. It ensures that only authorized individuals can access the system and perform actions.

Data Format: The Authentication System typically operates with structured data formats to handle user credentials securely.

Access Protocol:

  • HTTP/HTTPS: The system accepts authentication requests through HTTP or HTTPS protocols.

  • OAuth: The system may support OAuth protocol for delegated authentication.

Security: The Authentication System prioritizes the security of user credentials and sensitive information. It implements robust security measures such as secure communication protocols (e.g., HTTPS), and proper handling of passwords to protect against unauthorized access and data breaches.

Reliability: The system is reliable and available to handle user authentication requests consistently. It minimizes downtime and ensures that users can access the system whenever needed.

Performance: The Authentication System is designed to handle authentication requests efficiently, providing fast response times to users.

Notification System

The Email Notification System is responsible for sending out notifications and communications via email in the travel booking system. It facilitates communication between the system and users by delivering important updates, booking confirmations, and other relevant information through email channels.

The Email Notification System in the travel booking system operates with a structured data format for emails, often employing MIME. It communicates using standard email protocols like SMTP for sending and receiving emails.

Reliability: The Email Notification System ensures the reliable delivery of emails, minimizing the risk of message loss or failure.

Performance: The system is designed to handle a significant volume of email notifications efficiently. It optimizes the email delivery process to minimize latency and ensures timely notifications for users.

Security: The Email Notification System prioritizes the security and privacy of email communications. It incorporates measures such as encryption, secure connections (e.g., TLS/SSL) to protect sensitive information contained within the emails.

Database System

The Database is an external system used by the travel booking system to store and manage data related to touristic packages, customer information, bookings, and reports. It provides persistent storage for the travel booking system's data and ensures data integrity and reliability.

Data Format: Structured format (e.g., tables and fields in an RDBMS).
Access Protocol: SQL (Standard Query Language) or suitable API (e.g., ODBC, JDBC).
The Database in the travel booking system uses a structured data format, typically tables and fields in an RDBMS. It can be accessed and queried using SQL or an appropriate API like ODBC or JDBC.

Data Integrity: The Database ensures the accuracy and consistency of stored data, utilizing mechanisms such as data validation, constraints, and relational integrity rules.
Reliability: The Database is designed to be highly reliable, minimizing the risk of data loss or corruption through mechanisms such as backups, replication, and fault-tolerant configurations.
Performance: The Database is optimized to provide efficient and responsive data access and retrieval, supporting the expected workload of the travel booking system.
