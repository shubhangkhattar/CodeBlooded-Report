Travel Booking System : UML Class Diagram - Sprint 1

Created by Anushka Shetty, last modified on May 30, 2023

Design of the current product increment (class diagram)

User and TravelPackage: This relationship is represented as an association. A user can browse, select, and interact with multiple travel packages, and a travel package can be associated with multiple users. It is a typical association where both classes can exist and function independently.

User and TravelPackage (Dependency): In the context of booking a travel package, we can also identify a weak relationship between User and TravelPackage. The User class may depend on the TravelPackage class for booking purposes. However, this dependency is not explicitly shown in the class diagram provided since it is not part of the current sprint.

TravelPackage and Flight: TravelPackage can includeÂ  Flights as part of its offering. The aggregation relationship signifies that the Travel Package is composed of Flights. Each Flight represents a part of the overall Travel Package.

TravelPackage and Hotel: TravelPackage can include one or more Hotels as part of its offering. The aggregation relationship signifies that the Travel Package is composed of Hotels. Each Hotel represents a part of the overall Travel Package.

Travel Package and Activity: TravelPackage can include one or more Activities as part of its offering. The aggregation relationship signifies that the Travel Package is composed of Activities. Each Hotel represents a part of the overall Travel Package.
